# 「鹅苗雷达」实习生成长导航智能看板 — 方案说明

## 一、问题诊断

某部门迎来 20 名校招实习生，分布在研发、产品、销售等不同岗位。当前存在三大痛点：

**1. 导师带教凭经验，缺乏标准化节奏。** 导师各自为政，有的每周 1v1，有的一个月才聊一次。实习生遇到问题时，导师往往只能事后补救，无法提前发现风险信号。

**2. 实习生对"我该学什么"感到迷茫。** 新人不知道自己的成长进度是否正常，频繁私聊 HR 询问"我这样算不算正常"，HR 被动响应，效率低下。

**3. HR 追问适岗情况时缺乏数据支撑。** 招聘同学想知道"这批人最近怎么样"，只能逐个问导师，没有全局视角和量化指标。

## 二、方案设计

鹅苗雷达采用 **"数据采集 → 风险识别 → 智能建议 → 闭环干预"** 四阶段架构：

| 阶段 | 功能 | 技术实现 |
|------|------|----------|
| 采集 | 自动聚合周报、导师评价、任务完成率、情绪信号 | Prisma ORM + 多源数据接口 |
| 识别 | 五维风险模型（低投入/能力错配/融入/情绪压力/留用） | 加权评分算法 + 趋势分析 |
| 建议 | AI 生成 HR 行动建议、导师沟通模板、个性化成长方案 | LLM API + 结构化 Prompt |
| 干预 | 从风险发现到 HR 介入的完整闭环追踪 | 状态机 + 提醒机制 |

**核心页面：**
- Dashboard 总览：风险分布饼图、岗位对比柱状图、AI 今日提醒
- 预警池：按风险等级筛选，一键触发干预流程
- 实习生详情：能力雷达图、周报情感分析、导师评价时间轴
- AI 建议：HR 行动清单、导师沟通模板库、个性化成长方案
- 高潜人才：识别高潜力实习生，推荐重点培养
- 岗位对比：跨岗位横向对比适岗度、风险率、高潜率
- **我的成长（实习生视角）**：个人目标、90 天学习路径、技能雷达、导师反馈、周报记录

## 三、AI 工具选型理由

| 维度 | 选择 | 理由 |
|------|------|------|
| 风险评估 | 加权评分模型 | 五维风险模型可解释性强，每个风险信号都有明确来源，便于向导师和 HR 解释 |
| 情感分析 | LLM API | 周报文本的情感信号（焦虑、迷茫、消极）需要语义理解，规则引擎难以覆盖 |
| 建议生成 | LLM + 结构化 Prompt | 基于实习生画像生成个性化建议，比模板化输出更精准 |
| 数据可视化 | ECharts | 支持丰富的图表类型（饼图、柱状图、雷达图），交互性强，适合看板场景 |

**为什么选择混合架构（规则引擎 + LLM）：**
1. **可解释性优先：** 风险评分的核心逻辑是规则引擎，每个分数都能追溯到具体行为数据
2. **成本可控：** 高频查询（Dashboard、预警池）走规则引擎，低频但高价值的建议生成才调用 LLM
3. **容错性强：** LLM 调用失败时，系统仍能正常运行，有 fallback 机制兜底
4. **数据安全：** 实习生个人信息不离开服务器，仅脱敏后的摘要发送给 LLM

## 四、关键配置

- **风险评分权重：** 任务完成率（30%）、周报情感（20%）、导师评价（25%）、团队融入（15%）、出勤率（10%）
- **风险阈值：** 高风险 ≥ 70 分，中风险 40-69 分，低风险 < 40 分
- **高潜识别标准：** 适岗度 ≥ 80 且风险分 < 30 且连续 3 周正向成长
- **AI 提醒频率：** 每日自动扫描，高风险实习生实时预警
- **导师沟通模板：** 按风险类型分类（低投入/能力错配/融入/情绪/留用），每类 3-5 个问题模板

**Demo 亮点：** 
1. **一屏讲清全局** — Dashboard 汇总风险分布、岗位对比、AI 提醒，30 秒了解整体情况
2. **从发现到闭环** — 点击风险实习生 → 查看详情 → 触发干预 → AI 生成建议 → 导师执行，完整闭环
3. **多角色视角** — HR 看全局、导师看个案、实习生看成长，三个视角一套系统
